Task 5: operators Documentation

Operators

In this assignment we will refactor the Person and Marriage classes from the third assignment. We are going to add various operators.

Note
You'll have to copy Person.h, Person.cc, Marriage.h and Marriage.cc from task 3 (relations) to this directory.
The Person.h and Marriage.h files provided in this directory are just dummies to keep doxygen happy.
Do not overwrite any other existing files.

Assignment

  • Write a "helper", i.e. a stand-alone output operator (<<), to print a Gender constant (MALE, FEMALE) on an ostream (e.g. as 'm' or 'f' or as "male" and "female").
    Note: Since an enum is not a class there is no need to declare it a friend.
    Note: Since Gender is defined within class Person, you'll need to prefix Gender and the constants with Person:: e.g. Person::Gender, Person::MALE and Person::FEMALE.
    Hint: ternary operator: expr ? if_true : if_false .
  • Write a "helper", i.e. a stand-alone output operator, to print a Person object on an ostream. It should print the name, year of birth and the gender, e.g. "Beast (1970:m)", "Beauty (1976:f)".
    Note: It should be declared a friend of Person so it will be granted access to the private attributes of a Person.
  • Write a "helper", i.e. a stand-alone output operator, to print a Marriage object on an ostream. It should print the year they got married, the two people involved, and if they have a child, it should also print the child.
    Note: It should be declared a friend of Marriage so it will be granted access to the private attributes of a Marriage.
  • Write a "helper", i.e. a stand-alone "less-than" boolean operator, to compare two Persons by comparing their names.
    Note: The string class already provides a suitable < operator.
    Note: Such a < helper is used by the standard library when sorting Persons.
  • Write a "helper", i.e. a stand-alone "less-than" boolean operator, to compare two Marriages by comparing their wedding years.
    Note: Such a helper is used by the standard library when sorting Marriages.
Note
Your code will not compile until you have defined all five operators because your IDE insists on building the entire program all the time.

And example of possible test code:

// Create two people
Person* p1 = new Person("jan", 1900, Person::MALE);
Person* p2 = new Person("maria", 1910, Person::FEMALE);
// Marriage
Marriage* hp = new Marriage(p1, p2, 1929);
// Using the various operators you wrote
cout << (*p1) << endl; // information about the first person
cout << (*p2) << endl; // information about the second person
cout << (*hp) << endl; // tells the two people involved
check(*p1 < *p2); // verify: "jan" < "maria"
check(!(*p2 < *p1)); // verify: ! ("maria" < "jan")
// Add a child
Person* pc = new Person("joop", 1931, Person::MALE);
hp->addChild(pc);
cout << (*hp) << endl; // now also tells the child
// ... etc ...

The output should be something like:

jan (1900:m)
maria (1910:f)
married 1929: jan (1900:m) + maria (1910:f)
married 1929: jan (1900:m) + maria (1910:f) & joop (1931:m)

HINTS: about friends, the output-operator, the string compare operators, the ternary conditional-operator, the comparison operators.

For further study: about operators and operator overloading.
